Troubleshooting

Common issues and the fastest checks.

Docs home Platform status Skyforge home

I click a tool and get bounced back to the home page

  • This means you’re not logged in yet. The portal will route you to /status and prompt you to sign in.
  • Log in on the home page once (SSO), then your session is used to unlock tools.

Health is red / “down”

  • Check the live report: /status/summary
  • Check pod status: kubectl -n skyforge get pods
  • Quick check: kubectl -n skyforge get pods and curl -k https://<hostname>/status/summary

If you configured alerting, health changes can also be pushed to a webhook via SKYFORGE_ALERT_WEBHOOK_URL.

Git git push/pull fails

  • Use the repo URL under /git/<user>/<repo>.git.
  • If you’re using a non-browser git client, ensure you authenticate to Git (HTTP basic / PAT / SSH).

Resetting the stack

kubectl apply -k k8s/overlays/k3s-traefik-secrets
kubectl -n skyforge rollout restart deploy/skyforge-server
