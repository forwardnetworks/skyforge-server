Collectors

Collectors connect Skyforge labs to Forward SaaS for reachability tests and collections.

Open collectors page

What a collector is

  • One collector = one in-cluster Deployment + one Forward collector identity.
  • Collectors run connectivity tests (SSH) and optionally start full collection after a lab is configured.
  • You can create multiple collectors (useful when you need different Forward credentials).

Recommended workflow

  1. Create a collector named skyforge-<yourname>.
  2. Wait for Status to become Running and Forward to become Connected.
  3. Use that collector in deployments (for in-cluster Netlab/Clabernetes runs).

Troubleshooting

  • Stuck “Starting…”: open Logs and look for auth / DNS / routing issues.
  • Connectivity fails early: it may be a timing issue; retry after devices finish boot + post-up config applies.
  • SSH errors: verify the device is actually listening on port 22 (some NOS take time to start SSH).
