Labs

Skyforge supports two lab authoring styles: Netlab templates and a Containerlab-first designer.

Deployments Designer

Netlab

  • Best for reusable templates and “one file” topology definitions.
  • Skyforge runs Netlab to generate Containerlab/clabernetes artifacts.
  • Overrides are typically done via NETLAB_* environment variables.

Containerlab designer

  • Graphical, EVE-NG/GNS3-style workflow for building labs quickly.
  • Focuses on Containerlab/clabernetes (not Netlab).
  • Ideal for “import running lab and tweak” workflows.

Live topology tools

  • Graph view shows nodes/links for a running deployment.
  • Per-node: Terminal, Logs, Describe (depends on device type).
  • Linux endpoints may run “noise” to generate L2/L3 state.
