Security & secrets

What’s public, what’s gated, and where credentials live.

Docs home Skyforge home

Docker secrets

  • Internal credentials are stored as Docker secrets under ./secrets/ (gitignored).
  • Services read secrets from /run/secrets/… inside containers.
  • Bootstrap (dev defaults): populate ./secrets/ with local values (no committed defaults).
  • Terraform/OpenTofu state uses object storage bucket terraform-state with a scoped service account (object_storage_terraform_access_key/object_storage_terraform_secret_key).

What’s intentionally public

  • / (Skyforge landing)
  • /file-drop.html + /files/… (anonymous upload/download)
  • /status/summary (status feed, recommended)
  • /data/platform-health.json (legacy)
  • /docs/… (these docs)

What’s gated behind Skyforge login

  • /dashboard/deployments, /netbox/, /nautobot/, /git/, /minio-console/
  • When not logged in you’ll be redirected to /api/skyforge/api/oidc/login?next=….

Always-admin users

  • Set SKYFORGE_ADMIN_USERS (comma-separated usernames) to force admin flags across UIs.
  • The workspace sync interval is controlled by SKYFORGE_WORKSPACE_SYNC_SECONDS.
  • Admins can impersonate a user for troubleshooting; during impersonation Skyforge sets Remote-User to the impersonated user and adds X-Skyforge-Actor + X-Skyforge-Impersonating headers for audit/debug.
