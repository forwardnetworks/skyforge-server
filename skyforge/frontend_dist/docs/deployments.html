Deployments

Create a lab deployment from a blueprint template, run it, and inspect nodes/logs/topology.

Open deployments

Mental model

  • Deployment: the lab you intend to run (template + variables + settings).
  • Run: an execution attempt of that deployment (logs, artifacts, status).
  • Topology: the nodes and links created in Kubernetes (clabernetes).

Fast path

  1. Log in and verify your personal context is initialized.
  2. Create a deployment from a Netlab template.
  3. Click Validate to catch YAML/Netlab errors before running.
  4. Click Start.
  5. Open the run logs. Wait for “Topology is ready”, then post-up config.
  6. Open the graph to see the live topology, then Terminal/Logs/Describe per node.

Variables & overrides

  • Netlab supports NETLAB_* environment variables to override defaults (e.g. NETLAB_DEVICE).
  • Skyforge-specific knobs are passed as SKYFORGE_* variables.
  • Use Variable Groups for reusable sets of variables.

Common pitfalls

  • “Connectivity test failed” immediately: some NOS take time to start SSH. Retry / wait.
  • Template validate passes but run fails: verify you’re validating with the same variables you run with.
  • Device unreachable: check node readiness + collector reachability + SSH enabled.
