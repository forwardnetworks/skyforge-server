Blueprints (templates) via Git

Skyforge reads deployment templates (“blueprints”) from a shared Gitea repository. Admins manage it using normal Git.

Where Skyforge reads templates from

  • Repo: skyforge/blueprints (Gitea)
  • Folders at repo root: containerlab/, netlab/, terraform/
  • Skyforge UI loads templates from those folders; if the repo is empty you’ll see “failed to load templates”.

Seed / update blueprints (admin workflow)

  1. Log into Skyforge as the superuser and open Gitea (/git/).
  2. Ensure the repo exists: skyforge/blueprints (public is recommended for easy browsing via Explore).
  3. From your machine:
# Clone using the Skyforge superuser (example host)
git clone https://<host>/git/skyforge/blueprints.git
cd blueprints

# Copy templates into the correct directories
mkdir -p containerlab netlab terraform
# e.g. cp -R /path/to/my-topos/* netlab/

git add -A
git commit -m "Add blueprints"
git push

Troubleshooting

  • “failed to load templates”: verify the repo exists and has the expected folder layout.
  • Template not listed: ensure it’s committed to skyforge/blueprints and the file path is correct.
