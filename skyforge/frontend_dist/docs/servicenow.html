ServiceNow: Forward Connectivity Ticket Demo

Install the Forward “connectivity ticket” demo into a ServiceNow Personal Developer Instance (PDI).

What this demo does

  • Collects srcIp/dstIp/protocol/port from a Service Portal widget.
  • Calls Forward Path Search (GET /api/networks/{networkId}/paths).
  • Normalizes the result into allowed/blocked + hop trace.
  • Stores hop records on the ticket.

Prerequisites

  • A ServiceNow Personal Developer Instance (PDI).
  • Forward SaaS credentials (for https://fwd.app) via either:
    • a Skyforge Forward collector (recommended), or
    • manual username/password.

Setup

  1. Create a ServiceNow PDI.
  2. In Skyforge, open ServiceNow (left navigation).
  3. Enter the ServiceNow instance URL and admin creds.
  4. Select a Forward credential source (collector recommended).
  5. Click Save.
  6. If your PDI is sleeping, use PDI status → Wake up.
  7. Create the demo schema (manual step, in ServiceNow UI):
    • Navigate to System Definition → Tables and click New.
    • Create the ticket table:
      • Label: Forward Connectivity Ticket
      • Name: leave blank (ServiceNow auto-creates u_forward_connectivity_ticket)
      • Extends table: Task (recommended for number, short_description, assignment_group, work_notes)
    • Create the hop table:
      • Label: Forward Connectivity Hop
      • Name: leave blank (ServiceNow auto-creates u_forward_connectivity_hop)
    • Add the required columns (dictionary entries). Skyforge lists the exact required fields under ServiceNow → Schema status.
    • Fast path: after creating a table, open it and use the Columns related list to create the fields.
    • Alternate path: System Definition → Dictionary → New and set Table + Column name + Type.
    • Required field types (recommended):
      • u_src_ip, u_dst_ip: String
      • u_protocol: Choice (TCP/UDP)
      • u_dst_port: Integer
      • u_forward_network_id, u_forward_snapshot_id: String
      • u_allowed: True/False
      • u_analysis_status: Choice (not_run/running/complete/error)
      • u_analysis_error, u_raw_excerpt, u_forward_query_url: String
      • Hop table: u_ticket Reference → u_forward_connectivity_ticket, and Strings for u_device, u_ingress, u_egress, u_note, plus Integer u_hop_index
  8. Click Install demo app (installs scripts, REST message, portal widget, and properties).
  9. (Optional) Click Configure Forward (ticketing) to enable Forward’s built-in ServiceNow ticketing integration.

Notes

  • ServiceNow calls Forward SaaS directly. Skyforge only installs/configures ServiceNow artifacts.
  • Skyforge installs scripts/widgets/etc via ServiceNow’s Table API, but does not create the schema (custom tables/fields).
  • The installer overwrites demo artifact contents by name (scripts, widget, REST message, properties).
